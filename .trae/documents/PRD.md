# 盘前财经资讯研判智能体 — 产品需求文档 (PRD)

> 版本：v1.0 · 更新日期：2026-06-10

---

## 1. 产品定位

**Slogan**：每天 9:30 开盘前 5 分钟，看懂今日主战场。

**核心使命**：在前一交易日 15:00 收盘后到当日 9:30 开盘前之间，自动从财联社、东方财富、同花顺三大平台抓取"预测/观点"类财经快讯，识别其影响的板块，输出结构化盘前简报，并通过 Web 端 + 微信订阅让用户在开盘前 5 分钟内完成决策辅助阅读。

**核心约束**：
- 只看"自变量"（预测未来的机构研判、行情预测、热点推演、后市分析），"因变量"（已发生事实）仅作背景。
- 不构成投资建议，定位为信息聚合 + 辅助研判工具。
- 目标用户：短线/中线投资者、机构研究员助理、财经自媒体作者、交易员。

---

## 2. 目标用户

| 用户 | 痛点 | 期望 |
|------|------|------|
| **散户投资者** | 开盘前信息爆炸，筛选成本高 | 5 分钟内看完全部预测类要点 |
| **机构/私募研究员** | 需要快速跨平台聚合信息 | 自动归类、自动板块识别 |
| **财经自媒体** | 写作前需要素材 | 直接拿现成简报二次创作 |
| **订阅用户（付费/免费）** | 不想每天爬网页 | 微信 9:25 自动推送到手机 |

## 3. 核心功能

### 3.1 自动抓取
- **触发时机**：每日 8:30（可配置），覆盖前一交易日 15:00 → 当日 9:30 的窗口。
- **数据源**：
  - 财联社电报（通过 china-finance-rss 的新浪财经备份源）
  - 东方财富 7×24 快讯
  - 同花顺 7×24 快讯
- **去重**：标题相似度 > 80% 视为重复。
- **链接修复**：东方财富/同花顺详情页易失效，自动转为站内搜索链接。

### 3.2 智能分析
对每条资讯自动输出 5 维标签：
1. **板块归属**（12 大板块 + 子板块细分）
2. **资讯类型**（研报/政策/公司/数据/行业）
3. **自变量/因变量**（核心过滤：只看预测/观点类）
4. **情绪倾向**（利好/利空/中性）
5. **重要性评分**（1-10 分，≥7 分才进入精选）

### 3.3 结构化简报
输出符合 `output_spec.md` 模板的 Markdown 简报：
- 核心预判（机构共识/分歧/整体情绪）
- 热点板块推演（TOP 5 表格）
- 关键自变量资讯精选（TOP 10 表格）
- 风险提示

### 3.4 Web 应用
- **首页（今日速览）**：编辑型杂志风大字号标题 + 时间戳 + 倒计时 + 板块热度图 + 资讯流
- **简报页**：完整 Markdown 渲染版
- **订阅页**：极简表单（邮箱 + Server酱 SendKey + 板块选择）
- **历史页**：可回溯 7/30 天的历史简报

### 3.5 微信推送（订阅）
- 用户在 Web 端填写 **Server酱 SendKey**（免费版 5 条/天足够）
- 每天 9:25 自动推送当次简报摘要到个人微信
- 推送内容：核心预判 + TOP 3 板块 + TOP 3 资讯 + 风险提示

### 3.6 定时任务
- **执行时间**：每日 8:30（采集+分析+生成）+ 9:25（推送）
- **Windows 任务计划** 或 **后台常驻 schedule 进程**（用户二选一）
- **立即执行**：`启动助手.bat` 选项 3

## 4. 用户流程

```
[用户首次使用]
双击 启动助手.bat → 选 1（启动 Web 服务） → 浏览器打开 http://localhost:5000
→ 看到今日速览（首次为空，需手动刷新） → 点"立即刷新" 触发抓取
→ 阅读简报 → 切到"订阅"标签 → 填写 Server酱 SendKey → 订阅成功 → 微信收到测试消息

[每天定时运行]
8:30  Windows 任务计划自动跑 scheduler.py → 抓取+分析+生成
9:25  scheduler.py 内置定时再次触发推送 → Server酱推送到所有订阅者微信
```

## 5. 信息架构

```
首页 (/)
├── 顶部导航：今日速览 / 板块推演 / 订阅中心 / 历史档案
├── 头部区：盘前速览大字标题 + 实时时钟 + 开盘倒计时 + 版本号
├── 统计区：今日资讯数 / 涉及板块数 / 整体情绪 / 时间窗口
├── 板块热度图：12 宫格热力图，点击筛选
├── 资讯流：编辑型卡片，含板块标签/情绪/重要性/研判类型
└── 底部：免责声明

简报页 (/report)
└── 完整 Markdown 渲染版（5 个区块）

板块推演 (/sectors)
└── 12 个板块的详细分析卡片

订阅中心 (/subscribe)
└── 表单（邮箱、SendKey、板块多选、测试推送、订阅列表）

历史档案 (/history)
└── 时间线列表，点击查看历史简报
```

## 6. 设计原则（核心）

- **编辑型杂志感**：参考 Bloomberg Terminal × 华尔街日报 × Monocle Magazine
- **深色主题 + 烫金强调**：午夜深蓝底 + 米白文字 + 黄铜色点缀
- **大字号衬线标题 + 等宽数据**：Noto Serif SC 标题 + JetBrains Mono 时间戳
- **信息密度高但不杂乱**：栅格化 + 细金线分割
- **倒计时 + 实时时钟**：增加"金融终端"沉浸感
- **动效克制**：入场级联 + 数字滚动 + 倒计时脉动

## 7. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 单次抓取-分析-生成 ≤ 30 秒（数据源 3 个） |
| **可用性** | 一键启动（双击 bat），无需配置即可跑通 |
| **兼容性** | Windows 10/11，Python 3.8+ |
| **可观测** | 控制台彩色输出 + 日志文件 |
| **可扩展** | 模块化设计，新增数据源/推送渠道只需新增模块 |

---

## 8. 风险与限制

- 财联社官方 API 已失效，使用新浪财经备份源（仍可能受反爬影响）
- 东方财富/同花顺详情页易 404，已自动转为站内搜索链接
- Server酱免费版每天 5 条上限，盘前简报每天 1 条完全够用
- Windows 任务计划需要用户密码登录会话（不能完全无人值守）
